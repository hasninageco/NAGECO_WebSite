# Deploy Next.js Standalone Artifact (Style 2)

This mode avoids uploading the full frontend source. You upload only runtime artifact files.

## 1) Build artifact locally

Run in project root:

```bash
npm install
npm run artifact
```

After success, use folder:

- `dist-standalone/`

### Optional: Build artifact on cPanel (low-memory mode)

If your hosting can run Node scripts, use this in project root:

```bash
npm install --include=optional
npm run artifact:cpanel
```

Notes:

- `artifact:cpanel` uses a constrained build mode to reduce RAM usage.
- It sets `LOW_MEMORY_BUILD=1`, disables lint during build, limits workers, and caps Node old-space memory.
- If build still fails with out-of-memory, use local build (`npm run artifact`) and upload the generated artifact instead.

## 2) Upload artifact to cPanel

Upload the **contents** of `dist-standalone/` into your Node app root on cPanel (for example: `/home/<user>/nageco-site`).

The app root should contain at least:

- `server.js`
- `.next/static/...`
- `public/...`
- `node_modules/...` (minimal runtime modules generated by Next standalone)

If `.next/static` is missing after upload, styles and client JS will not load.

## 3) Setup Node.js App

In cPanel Setup Node.js App:

- Application root: your upload root (example `nageco-site`)
- Application startup file: `server.js`
- Mode: production

Set environment variables (minimum):

- `NODE_ENV=production`
- `HOST=0.0.0.0`
- `PORT=3000`
- `DATABASE_URL=...`
- `DIRECT_URL=...`
- `NEXTAUTH_URL=...`
- `NEXTAUTH_SECRET=...`
- `BACKEND_API_ORIGIN=https://nageco.com/nageco-back`

Optional temporary maintenance lock (password prompt for all pages/routes):

- `SITE_LOCK_ENABLED=1`
- `SITE_LOCK_USERNAME=<temporary-username>`
- `SITE_LOCK_PASSWORD_SOURCE=database` (uses hashed admin password from DB)
- `SITE_LOCK_PASSWORD=<temporary-password>` (used only when `SITE_LOCK_PASSWORD_SOURCE=env`)
- `SITE_LOCK_COOKIE_VALUE=<lock-version>` (change value to force all browsers to re-enter the password)

Restart app.

## 4) Verify

Open your domain and check key pages/admin/API routes.

If runtime fails, re-upload `dist-standalone/` contents and restart the app.
